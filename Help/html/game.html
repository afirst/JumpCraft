Scripting Object: Game Up

 

 
The Game object’s members can be accessed by simply typing Game. in the scripting editor.  When you type Game., the Scripting Editor pops up an AutoComplete list which shows all accessible PlatformStudio.psGame members.  Here is an example:

Some major objects insidee Game include Actions, cam, CurMap, Drawing, GameProperties, maps, and PresetActions.  Note: All arrays start at an index of 1. Index 0 is a blank item.

Game.Actions
Game.Actions is used to lookup or modify anything that can be modified in the Action Editor.  Using Game.Actions, you can access the game’s counters (Game.Actions.Counters) and their current values (Game.Actions.Counters(index).Value), the game’s timers (Game.Actions.Timers), the game’s actions (Game.Actions.Actions), and more. However, there are limitations using scripting. Since the actions compile to a DLL, calling Game.Actions.AddAction or modifying Game.Actions.Actions will have no effect on your game at all, except adding a new entry to the array of actions.  However, you can still modify the counters and timers, since they are stored internally.

Game.Cam
This gets the current viewport, which is set automatically.  Therefore, you should not modify this object.

Game.CurMap
This gets or sets information about the current map the user is playing. Some commonly used members include
Background
Cells
Clone
loc
MapHeight
MapName
MapWidth
Music
ResizeMap
Volume.

Game.CurMap.loc
This is the location layer for the current map.  Its commonly used members include
AddLocation
Locations
NumLocations
RemoveLocation.

Game.Drawing
This object contains drawing functions, such as drawing points, lines, boxes, text, and sprites, as well as information about the loaded fonts and images. Use RelativeToCam to control whether the coordinates of what you want to draw is relative to the camera (i.e. scrolls with the map), or not (i.e. fixed on the screen, such as a score counter, or a button).

Game.GameProperties
This object contains information about the game’s name, its version, copyrights, resolution, window options, and more.

Game.maps
This is the array of maps. Each map’s members are the same as that of Game.CurMap.  Here is an example usage:

‘Sets the last map’s name to “Final Level”
Game.maps(Game.NumMaps).MapName = “Final Level”

Game.PresetActions
This is the most basic class for scripting, and was built only for the purpose of scripting and its related actions.  You can use these functions to provide the same effect if you were to use the built-in actions, except you have more control over the parameters.  For instance, you can base them on variables. It is also convenient if your trigger’s action is mostly code, and you want to conveniently stick a built-in action in the middle of your code. Here is an example usage:

‘This code lets you start out with an invincibility
‘(after getting hurt) of 4 seconds.
‘If you collect a special item, 2 more seconds
‘will be added to your invincibility duration.

‘This code is in Class View
Dim HurtTime As Integer = 4

‘The rest of the code is Procedure view code,
‘with the function name being the description
‘of the trigger
Function [CollectSpecialItem]()
   'Allows you to be invincible for 2 more seconds
   HurtTime += 2
End Function

Function [DecreaseHealth]()
   Game.PresetActions.Hurt(HurtTime, "Fade", 0.2)
End Function
